Build a working MVP demo web application called “Tarixchi AI”.

The app must be a clickable, polished, demo-ready MVP for an AI-powered history learning ecosystem for Uzbek school students.

====================================================
PRODUCT CONCEPT
====================================================

Project name:
Tarixchi AI

Slogan:
“Tarixni yodlama. Tarixchi kabi o‘yla.”

One-line explanation:
Tarixchi AI is an interactive AI-powered history learning ecosystem for school students who want to self-study history in a more engaging, visual, and personalized way.

Problem:
Many students study history by memorizing dates, names, and events. Extra history tutoring is becoming common, but much of the tutoring process is repetitive: the teacher explains some information, asks questions, checks whether the student read the material, and gives tests. This process can be boring and expensive.

Solution:
Tarixchi AI automates the repetitive part of the history teacher’s work:
- explains topics through storytelling,
- answers student questions during the lesson,
- shows timelines and cause-effect visuals,
- gives quizzes and image-based questions,
- tracks student progress,
- creates a personalized self-study experience.

Target user:
Uzbek school students, especially 7th–11th grade students who are learning history independently or preparing for tests.

Main MVP demo user journey:
1. User opens Tarixchi AI.
2. User sees a modern dashboard.
3. User goes to “Darsliklar”.
4. User selects grade and topic.
5. User opens “Jadidchilik harakati”.
6. AI teacher explains the topic through storytelling.
7. User can click “Nega muhim?”, “Oddiyroq tushuntir”, “Timeline ko‘rsat”, “Testga tayyorla”.
8. User sees an interactive timeline.
9. User sees a cause-effect map.
10. User takes a quiz.
11. Quiz score is saved in profile using localStorage.
12. User can try “Interviews”, “Questions”, and “Threads” demo sections.
13. User profile shows progress, quiz score, strengths, weaknesses, and completed topics.

AI API ROUTE REQUIREMENTS
====================================================

Create /api/ask-history/route.ts.

Input:
{
  "question": string,
  "topic": string,
  "grade": string
}

Output:
{
  "answer": string,
  "followUpQuestion": string,
  "mode": "ai" | "fallback"
}

Logic:
- If process.env.GEMINI_API_KEY exists, call Gemini API.
- If key missing or API fails, return fallback answer.
- Never expose API key to frontend.
- Never crash.
- Always return valid JSON.
