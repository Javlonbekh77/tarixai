You are a senior backend engineer and AI integration architect.

Build the backend AI layer for “Tarixchi AI”, an AI-powered interactive history lesson platform for Uzbek school students.

The goal:
Admin provides source files or source text about a history topic.
The backend analyzes the material using free or free-tier AI providers and generates structured lesson data for the frontend lesson page.

This is an MVP backend.
Do not overbuild.
The backend must work even if API keys are missing.
Use provider fallback:
1. Gemini API
2. Groq API
3. OpenRouter API
4. Ollama local
5. Mock fallback

====================================================

Product:
Tarixchi AI

Interactive lesson page.

Provider interface:

  name: string;
  isAvailable: () => boolean;
  generateText: (params: GenerateTextParams) => Promise<GenerateTextResult>;
  generateLessonJson?: (params: GenerateLessonParams) => Promise<LessonJson>;
}

aiRouter behavior:
- Try Gemini first.
- If Gemini key missing or request fails, try Groq.
- If Groq key missing or request fails, try OpenRouter.
- If OpenRouter key missing or request fails, try Ollama local.
- If Ollama fails, return mock fallback.
- Always return valid JSON response.
- Never crash.
- Include providerUsed in every response.

====================================================
WHY EACH PROVIDER

Gemini:
Use as primary provider for:
- analyzing source text

- structured JSON lesson output

Groq:
Use as fast fallback for:
- student Q&A

OpenRouter:

Use as local fallback:

Mock:
Use when all AI providers fail.

====================================================
REQUIRED API ENDPOINTS
====================================================

1. POST /api/admin/generate-lesson

Purpose:
Admin gives source content. Backend generates a full structured lesson JSON.

Input:
{
  "topic": "Jadidchilik harakati",
  "grade": "8-sinf",
  "sourceText": "....",
  "language": "uz-Latn"
}

Behavior:
- Validate input.
- Use Gemini if available.
- Ask AI to transform sourceText into structured lesson JSON.
- If Gemini fails, try fallback providers.
- If all fail, return default demo lesson JSON.
- Response must include:
  {
    "success": true,
    "providerUsed": "gemini" | "groq" | "openrouter" | "ollama" | "mock",
    "lesson": LessonJson
  }

Important:
Do not invent facts.
If source is missing, write "source needed".
Output must match LessonJson schema.

2. POST /api/lesson/ask

Purpose:
Student pauses lesson and asks question. Timer pauses on frontend.

Input:
{
  "lessonId": "jadidchilik-harakati",
  "sceneId": 3,
  "question": "Jadidlar nega ta'limdan boshlagan?",
  "studentLevel": "slow" | "normal" | "fast",
  "currentSceneContext": "..."
}

Behavior:
- Answer in Uzbek Latin.
- Adapt answer to studentLevel.
- If slow: simpler explanation, analogy, shorter sentences.
- If normal: normal school-level answer.
- If fast: deeper answer with cause-effect and challenge question.
- Return:
  {
    "answer": "...",
    "followUpQuestion": "...",
    "timerAction": "pause",
    "providerUsed": "...",
    "suggestedNextAction": "continue_lesson" | "show_timeline" | "review_notes"
  }

3. POST /api/progress/update

Purpose:
Save/update student learning progress and classify learning speed.

Input:
{
  "userId": "demo-user",
  "lessonId": "jadidchilik-harakati",
  "sceneId": 4,
  "expectedSceneTimeSeconds": 120,
  "actualTimeSpentSeconds": 180,
  "pauseCount": 2,
  "questionCount": 1,
  "microQuestionCorrect": false,
  "quizScore": 70
}

Behavior:
- Calculate learningSpeed:
  fast if actualTimeSpent < expected * 0.7 and quizScore >= 80
  slow if actualTimeSpent > expected * 1.3 or quizScore < 60
  otherwise normal
- Return adaptive recommendation:
  slow → "Oddiyroq tushuntirish va ko‘proq misollar berish"
  normal → "Standart darsni davom ettirish"
  fast → "Challenge savollar va chuqurroq tahlil berish"

Return:
{
  "learningSpeed": "slow" | "normal" | "fast",
  "adaptiveRecommendation": "...",
  "nextContentMode": "simplified" | "standard" | "advanced"
}

4. POST /api/notes/generate

Purpose:
Generate personalized AI notes after lesson.

Input:
{
  "lessonId": "jadidchilik-harakati",
  "lessonSummary": "...",
  "studentProgress": {
    "learningSpeed": "slow",
    "quizScore": 65,
    "weakAreas": ["sabab-oqibat", "timeline"]
  }
}

Output:
{
  "providerUsed": "...",
  "notes": {
    "keyTakeaways": [],
    "examTips": [],
    "weakPointsReview": [],
    "oneSentenceSummary": "",
    "recommendedNextStep": ""
  }
}

5. POST /api/interview/ask

Purpose:
Ask question to a historical persona simulation.

Input:
{
  "persona": "Mahmudxo'ja Behbudiy",
  "topic": "Jadidchilik harakati",
  "question": "Nega ta'lim siz uchun muhim bo'lgan?"
}

Behavior:
- Return simulated answer based on historical context.
- Must include disclaimer:
  "Bu tarixiy shaxs asosidagi AI simulyatsiya."
- Do not claim it is the real person's actual words.
- Use Uzbek Latin.

Output:
{
  "disclaimer": "...",
  "answer": "...",
  "providerUsed": "..."
}

====================================================
IMPLEMENT PROVIDERS
====================================================

Gemini provider:
- Use fetch or official SDK if already installed.
- Endpoint/model can be configurable.
- Suggested model env:
  GEMINI_MODEL=gemini-2.5-flash
- For structured lesson generation, request JSON.
- If JSON parse fails, try to extract JSON safely.
- If still fails, throw provider error so aiRouter tries next provider.

Groq provider:
- Use OpenAI-compatible chat completions if simple.
- Env:
  GROQ_API_KEY
  GROQ_MODEL=llama-3.3-70b-versatile or current available default
- Good for text answers and fallback lesson generation.

OpenRouter provider:
- OpenAI-compatible API.
- Env:
  OPENROUTER_API_KEY
  OPENROUTER_MODEL=openrouter/free
- Use as fallback.
- Include HTTP-Referer and X-Title headers if needed.

Ollama provider:
- Env:
  OLLAMA_BASE_URL=http://localhost:11434
  OLLAMA_MODEL=gemma3
- Use /api/chat or /api/generate.
- Timeout quickly, for example 15 seconds, so app does not hang if Ollama is not running.

Mock provider:
- Always available.
- Return a complete Jadidchilik demo LessonJson.
- Use realistic Uzbek Latin content.
- This ensures MVP always works.
